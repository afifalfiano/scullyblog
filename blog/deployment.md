# Deployment

## Pengertian Cloud Computing
Cloud Computing adalah penyedia layanan untuk sumber daya system komputer khususnya penyimpanan data (cloud storage) dan daya komputasi tanpa harus langsung memanagement dari user. Istilah umum ini biasanya untk mendeskripsikan pusat data yang tersedia untuk banyak user melalui internet. 

Cloud mungkin saja terbatas untuk satuan organisasi (enterprise clouds) atau tersedia untuk banyak organisasi (public cloud). Cloud computing bergantung pada seberapa besar sumber daya yang dibagikan untuk mencapai koherensi dan skala ekonomi.

Para pendukung cloud public dan hybrid mencatat bahwa cloud computing memungkinkan perusahaan untuk menghindari biaya yang terlalu besar atau meminimalkan biaya infrastruktur di bidang TI. Para pendukung juga mengklaim dengan cloud computing sebuah perusahan dapat tetap menyalakan aplikasi dan berjalan cepat dengan pengembangan yang termanagement dan sedikit maintenance, dan memungkinkan tim IT untuk lebih cepat menyesuaikan sumber daya untuk memenuhi permintnaan yang berfluktuasi dan tidak dapat diprediksi. 

Cloud computing menggunakan kategori “pay-as-you-go” yang artinya kita membayar sesuai apa yang kita gunakan. Ketersediaan kapasitas jaringa yang tinggi, biaya komputer dan penyimpanan yang terjangkau dan cukup baik ini serta mengadopsi hardware virtualization, service-oriented architecture dan utility computing yang telah mendorong pertumbuhan cloud computing.

## Sejarah Cloud Computing
Cloud computing dipopulerkan oleh amazon dengan rilisnya produk elastic compute cloud pada 2006. Referensi dari frase cloud computing muncul pada awal tahun 1996 dimana untuk pertama kalinya terdapat pada dokumen internal Compaq.

• Pada Agustus 2006 Amazon membuat anak perusahan bernama Azmaon Web Service (AWS) dan memperkenalkan layanan Elastic Compute Cloud (EC2).

• Pada April 2008, Google merilis versi beta dari Google App Engine.

• Pada awal 2008, Nasa’s OpenNebula, proyek yang didanai RESERVOID European Commision menjadi perangkat lunak open source untuk pertama kalinya menyebarkan cloud private dan hybrid.

• Pada 2008, U.S National Science Foundation memulai program explorasi Cluster untuk mendanai riset akademik menggunakan teknologi Google-IBM cluster untuk menganalisis data dalam jumlah besar.

• Pada Ferbruari 2010, Microsoft merilis produk Microsoft Azure yang mana telah diumumkan sebelumnya pada Oktober 2008.

• Pada Juli 2010, Rackspace Hosting dan Nasa ikut meluncurkan open-source cloud-software initiative yag dikenal dengan OpenStack.

• Pada 1 Maret 2011, IBM mengumumkan IBM SmartCLoud framework untuk mensupport Smarter Planet, Diantara variasi komponen dari Smarter Computing foundation, cloud computing menjadi yang paling disorot.

• Pada 7 Juni 2012, Oracle mengumumkan Oracle Cloud. Cloud ini untuk pertama kalinya menyediakan pengguna dapat mengakses untuk mengintegrasika IT Solution termasuk SaaS, PaaS, dan IaaS.

• Pada Mei 2012, Google Compute Engine dirilis dalam bentuk preview, sebelum akhirnya benar-benar rilis secara umum pada Desember 2013.

• Pada 2019, terungkap bahwa Linux paling banyak digunakan di Microsoft Azure.

## Karakteristik Cloud Computing

Cloud computing menggunakan konsep dari Utility Computing untuk menyediakan metric sesuai dari servis yang digunakan. Cloud computing mecoba mengatasi masalah QoS dan keandalan model cloud grid lainnya. Berikut ini karakteristik dari cloud computing:

    • Ketangkasan untuk mengembangkan sesuai kebutuhan secara fleksibel seperti menambahkan resource atau memperluas resource infrastruktur.
    
    • Pengurangan biaya yang telah diklaim oleh penyedia layanan cloud.
    
    • Perangkat yang independen dan menyesuaikan lokasi
    
    • Perawatan cloud computing yang mudah.
    
    • Multitenancy yang mengizinkan untuk berbagi resource dan harga.
    
    • Performa yang selalu dimonitor
    
    • Produktivitas mungkin akan meningkat ketika bebrapa user dapat bekerja sama pada data yang sama secara bersamaan, daripada menunggu untuk disimpan dan dikirim melalui email.
    
    • Scalabilitas dan Elastis 
    
    • Keamanan dapat ditingkatkan untuk sentralisasi data.

#### The National Institeu of Standars and Technology mendifinisikan “5 essential characteristics”

    • On-demand self-service.
    
    • Broad network-access
    
    • Resource pooling.
    
    • Rapid elasticity.
    
    • Measured service.

## Service Model

Cloud computing berkonsep semua adalah layanan. Cloud computing menyediakan servis untuk model yang berbeda dan terdapat 3 standar model untuk per NIST seperti Infrastructure as a Service(SaaS), Platform as a service(PaaS), dan Software as a service(SaaS).

1. IaaS

    Infrastrucure as as Service mengacu pada layanan online yang menyediakan API tingkat tinggi yang digunakan untuk mengabstraksi berbagai detail dari tingkat rendah infra seperti sumber daya komputasi fisik, lokasi, partisi data, penskalaan, keamanan, cadangan dan lain-lain. Hypervisor menjalankan virtual mesin sebagai tamu. Kontainer Linux berjalan dalam partisi terisolasi dari satu kernel Linux yang berjalan langsung di perangkat keras fisik. Containerisasi menawarkan kinerja yang lebih tinggi daripada virtualisasi karena tidak ada overhead hypervisor

    Penyedia cloud IaaS memasok sumber daya sesuai permintaan dari client dengan mengambil resource yang terdapat di pusat data mereka. Untuk konektivitas area luas, pelanggan dapat menggunakan Internet atau cloud operator (jaringan pribadi virtual khusus).

2. Paas

    The NIST mendefinisikan PaaS dari cloud computing sebagai berikut. Untuk mendeploy kedalam infrastruktur cloud. Konsumen tidak mengelola atau mengontrol infrastruktur cloud yang mendasarinya termasuk jaringan, server, sistem operasi, atau penyimpanan, tetapi memiliki kendali atas aplikasi yang diterapkan dan mungkin pengaturan konfigurasi untuk lingkungan hosting aplikasi. 

    Vendor PaaS menawarkan lingkungan pengembangan untuk pengembang aplikasi. Penyedia biasanya mengembangkan perangkat dan standar untuk pengembangan dan saluran untuk distribusi dan pembayaran. Dalam model PaaS, penyedia cloud menghadirkan platform komputasi, biasanya termasuk sistem operasi, lingkungan eksekusi bahasa pemrograman, database, dan server web.

3. SaaS

    The NIST mendefinisikan SaaS dari cloud computing sebagai berikut. Kemampuan yang diberikan kepada konsumen adalah menggunakan aplikasi penyedia yang berjalan pada infrastruktur cloud. Aplikasi dapat diakses dari berbagai perangkat klien melalui antarmuka klien tipis, seperti browser web (mis., Email berbasis web), atau antarmuka program. Konsumen tidak mengelola atau mengontrol infrastruktur cloud yang mendasari termasuk jaringan, server, sistem operasi, penyimpanan, atau bahkan kemampuan aplikasi individu, dengan kemungkinan pengecualian pengaturan konfigurasi aplikasi khusus pengguna yang terbatas.

4. MBaaS

    Mobile backend as a service. Pemrograman aplikasi antarmuka (API) yang diekspos ke aplikasi mereka dan kit pengembangan perangkat lunak khusus (SDK). Layanan termasuk manajemen pengguna, pemberitahuan push, integrasi dengan layanan jejaring sosial dan banyak lagi.

5. Serverless Computing

    Komputasi tanpa server adalah model eksekusi kode komputasi awan di mana penyedia awan sepenuhnya mengelola memulai dan menghentikan mesin virtual sebagaimana diperlukan untuk melayani permintaan, dan permintaan ditagih oleh ukuran abstrak sumber daya yang diperlukan untuk memenuhi permintaan, bukan per mesin virtual, per jam.

6. FaaS

Function as a service adalah sebuah service untuk meremote sesuai panggilan yang dihosting layanan yang memanfaatkan komputasi tanpa server untuk memungkinkan penyebaran fungsi individu di cloud yang berjalan sebagai respons terhadap peristiwa.

#### Model Deployment

Terdapat 3 tipe utama pada model deployment ini:

1. Private Cloud

    Cloud pribadi adalah infrastruktur cloud yang dioperasikan hanya untuk satu organisasi, baik dikelola secara internal atau oleh pihak ketiga, dan dihosting baik secara internal maupun eksternal.

2. Public Cloud
   
   Disebut public cloud ketika layanan diberikan melalui jaringan yang terbuka untuk penggunaan umum. Layanan cloud publik mungkin gratis. Secara teknis mungkin ada sedikit atau tidak ada perbedaan antara arsitektur cloud publik dan privat, namun, pertimbangan keamanan mungkin berbeda secara substansial untuk layanan (aplikasi, penyimpanan, dan sumber daya lainnya) yang disediakan oleh penyedia layanan untuk audiens publik dan ketika komunikasi tersedia.
    
3. Hybrid Cloud

   Cloud hibrida adalah komposisi dari public cloud dan private cloud, seperti private cloud atau sumber daya lokal, yang tetap merupakan entitas yang berbeda tetapi terikat bersama, menawarkan manfaat dari beberapa model penerapan. Cloud hibrida juga dapat berarti kemampuan untuk menghubungkan layanan kolokasi, terkelola dan atau khusus dengan sumber daya awan.
   
Untuk jenis lainnya seperti community cloud, distributed cloud, multicloud, poly cloud, big data cloud dan hpc cloud.

## Arsitektur


Arsitektur cloud computing biasanya melibatkan beberapa komponen cloud yang saling berkomunikasi satu sama lain melalui mekanisme penggandengan yang longgar seperti antrian pesan. Ketentuan elastis menyiratkan kecerdasan dalam penggunaan kopling ketat atau longgar sebagaimana diterapkan pada mekanisme seperti ini dan lainnya.

####Keamanan dan Privasi

Pengguna cloud memiliki beberapa masalah privasi karena penyedia layana cloud bisa saja mengakses data yang disimpan di cloud sewaktu-waktu. Banyak penyedia cloud dapat berbagi informasi dengan pihak ketiga jika diperlukan untuk tujuan hukum dan ketertiban tanpa surat perintah. 

Solusi untuk privasi termasuk kebijakan dan undang-undang serta pilihan pengguna akhir tentang bagaimana data disimpan. Pengguna dapat mengenkripsi data yang diproses atau disimpan di dalam awan untuk mencegah akses yang tidak sah.

Sistem manajemen identitas juga dapat memberikan solusi praktis untuk masalah privasi dalam cloud computing. Sistem ini membedakan antara pengguna resmi dan tidak sah dan menentukan jumlah data yang dapat diakses oleh setiap entitas. Sistem bekerja dengan membuat dan mendeskripsikan identitas, merekam aktivitas, dan membuang identitas yang tidak terpakai.

Menurut Cloud Security Alliance, tiga ancaman teratas di cloud adalah Antarmuka dan API yang Tidak Aman, Kehilangan & Kebocoran Data, dan Kegagalan Perangkat Keras  yang masing-masing menyumbang 29%, 25%, dan 10% dari semua pemadaman keamanan cloud.

Ada beberapa kelemahan nyata dalam infrastruktur cloud yang membuat bisa saja untuk dimasuki orang jahat. Karena data dari ratusan atau ribuan perusahaan dapat disimpan di server cloud yang besar, peretas secara teoritis dapat mengontrol penyimpanan informasi yang sangat besar melalui satu serangan yang disebut dengan "hyperjacking"

Beberapa contohnya termasuk pelanggaran keamanan Dropbox, dan kebocoran iCloud 2014. Dropbox telah dibobol pada Oktober 2014, memiliki lebih dari 7 juta kata sandi penggunanya dicuri oleh peretas dalam upaya untuk mendapatkan nilai uang darinya oleh Bitcoin (BTC). Dengan memiliki kata sandi ini, mereka dapat membaca data pribadi serta data ini diindeks oleh mesin pencari (membuat informasi menjadi publik).
